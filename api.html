API Reference

Built for developers

Three endpoints cover the whole flow — submit, check status, and track credits. REST in, JSON out.

BASE URL

https://api.linkindexer.ai/v1

AUTHENTICATION

Every request needs a bearer token in the Authorization header. Find your key under Settings → API Keys.

POST /v1/submit

Submit URLs

Accepts an array of up to 1,000 URLs per request. Each valid URL consumes one credit.

curl -X POST https://api.linkindexer.ai/v1/submit \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"urls": ["https://example.com/new-page", "https://example.com/blog/post-1"]}'
const res = await fetch("https://api.linkindexer.ai/v1/submit", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${process.env.API_KEY}`,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    urls: ["https://example.com/new-page"]
  })
});
const data = await res.json();
import requests

response = requests.post(
    "https://api.linkindexer.ai/v1/submit",
    headers={"Authorization": f"Bearer {API_KEY}"},
    json={"urls": ["https://example.com/new-page"]}
)
data = response.json()

GET /v1/status/:batch_id

Check submission status

Poll a batch to see how many URLs are queued, processing, indexed, or failed.

curl https://api.linkindexer.ai/v1/status/batch_9f8a2c \
  -H "Authorization: Bearer $API_KEY"

EXAMPLE RESPONSE — 200 OK

{
  "batch_id": "batch_9f8a2c",
  "queued": 0,
  "processing": 1,
  "indexed": 1,
  "failed": 0
}
GET /v1/credits

Check credit balance

Useful to check before running a large batch job.

curl https://api.linkindexer.ai/v1/credits \
  -H "Authorization: Bearer $API_KEY"

EXAMPLE RESPONSE — 200 OK

{
  "plan": "pro",
  "credits_remaining": 8418,
  "credits_total": 10000,
  "renews_at": "2026-08-01T00:00:00Z"
}
